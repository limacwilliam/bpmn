HIT Website is a Next.js 15 App Router marketing platform with a country-locale i18n model (pt-br, es-co, en-us, etc.), a Supabase-backed CMS/admin, and a dual UI architecture:

6. REGRAS DE PADRONIZAÇÃO

Visual rules

Always use semantic tokens — bg-accent, text-primary, text-muted-foreground. Never hardcode #F15A22 in new code; use bg-accent or bg-cyan-hover.

Marketing CTAs are pills — rounded-full, not shadcn rounded-md.

Headings are black weight — font-black for H1/H2 on marketing; shadcn defaults for admin UI.

Section rhythm — wrap every section in .section-padding + .container-hit.

Cards use rounded-2xl on marketing; rounded-lg in shadcn/admin.

Hover = accent — links and cards shift to accent color on hover.

Warm neutrals only — no cold blue-gray; all grays have brown tint (HSL hue ~30).

Code rules

Server-first — pages are Server Components; extract 'use client' only when needed.

Client suffix — interactive page sections → {Name}Client.tsx.

i18n everywhere — getTranslations (server) or useTranslations (client); no hardcoded strings.

SEO on every page — generateMetadata + JSON-LD.

Loading states — every route gets a loading.tsx skeleton.

cn() for class merging — always use cn() from @/lib/utils.

Path alias @/ — never relative imports beyond one level.

Component rules

Marketing ≠ shadcn Button — use Tailwind pill links for CTAs.

shadcn for admin/forms — Button, Input, Dialog, Toast in admin and cookie consent.

No framer-motion on marketing — CSS animations only (except lightbox).

Group hover pattern — group on parent, group-hover: on children.

Responsive rules

Mobile-first — base styles for mobile, md: for desktop.

Primary breakpoint md: (768px) — nav toggle, grid columns, typography scale.

Hero typography — always scale: text-4xl md:text-5xl lg:text-6xl.

CTA rows — flex-col sm:flex-row for button groups.
